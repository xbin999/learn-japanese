# 📄 日语真实表达系统 — 页面输入同步工具需求文档 v3.0

## 一、目标

为日语真实表达成长系构建一个轻量输入工具，实现：

1. 用户将与 AI 讨论后的结构化文本直接粘贴到页面
2. 系统解析文本字段
3. 自动写入 **Notion "日语表达记录"数据库**
4. 支持可扩展的字段解析
5. 自动生成小红书分享图片

> 目标是保持操作简洁、轻量、快速完成沉淀，同时提供美观的分享图片生成功能。

---

## 二、页面功能需求

系统拆分为三个独立页面：

### 0️⃣ API 连通性验证（测试程序）
* **目的**：在改页面功能前，先验证 Gemini 1.5 Flash / 智谱 glm-4-flash-250414 API 可正常返回 JSON 结构
* **输入**：一段“AI 生成结构化文本”示例
* **输出**：严格符合“JSON结构”的 JSON 字符串，并可被 `JSON.parse` 成功解析
* **密钥**：通过环境变量 `GEMINI_API_KEY` / `ZHIPU_API_KEY` 注入，严禁硬编码
* **结果**：控制台输出解析后的 JSON，并验证必要字段存在

### 页面 1：输入与同步 (index.html)

#### 1️⃣ 输入区域
* **文本框**：多行输入，支持Ctrl+A全选，右上角提供"清除"按钮一键清空
* **结构化文本输入**：粘贴 AI 生成的结构化文本（非 JSON）
* **模型选择**：可选择 Zhipu / Gemini，默认本地使用 Zhipu，生产可切换 Gemini
* **学习者昵称**：单行输入，浏览器记录当前昵称，用于同步到 Notion 的“学习者”字段
* **提示文本**：可显示"请粘贴 AI 讨论后的结构化文本"
* **同步按钮**：先调用模型接口生成 JSON，再写入 Notion
* **过程提示**：显示当前处理步骤（AI 解析中 / Notion 同步中）
* **导航**：提供跳转到"查询与分享"、"导出复盘"页面的链接
* **视觉一致性**：与其他页面保持同一视觉风格（纸感背景、按钮风格、卡片边角、字体层级）
* **移动端便利**：按钮全宽、输入区自适应高度、触摸反馈清晰、主要操作无需横向滚动

#### 2️⃣ 文本解析与同步逻辑
* **解析输入**：结构化文本 → 调用 AI 转为 JSON
* **无预览**：不再展示解析预览，仅保留同步结果提示
* **同步逻辑**：AI 转 JSON 成功后写入 Notion 数据库，并写入“学习者”
* **解析容错**：缺失字段显示为空，不阻断提交
* **数据完整**：AI 输出缺失“我想表达”时，从原始结构化文本中自动补全

#### 3️⃣ JSON结构
```
{
  "title": "",
  "topic": "",
  "intent": "",
  "versions": { "v1": "", "v2": "", "v3": "", "v4": "" },
  "final": "",
  "coreStructure": "",
  "improvement": "",
  "errors": [{ "name": "", "description": "", "correctPattern": "" }],
  "vocab": [{ "word": "", "reading": "", "meaning": "", "example": "" }],
  "summary": "",
  "shareTitle": ""
}
```

### 页面 2：查询与分享 (share.html)

#### 1️⃣ 历史记录查询
* **默认展示**：进入页面自动加载最近 10 条 Notion 记录
* **加载更多**：支持点击加载更多记录
* **学习者昵称**：单行输入，浏览器记录当前昵称
* **学习者筛选**：仅查询“学习者”与昵称一致的记录
* **交互**：点击记录后，在右侧（或下方）显示详细预览和生成选项
* **导航**：提供跳转回"输入与同步"与"导出复盘"页面的链接
* **视觉一致性**：与输入页在配色、按钮、卡片、排版密度上保持统一
* **移动端便利**：两栏在手机自动堆叠，卡片可点击区域充足，图片预览自适应

#### 2️⃣ 小红书文案生成
* **模板机制**：支持通过独立的模板文件（如 `templates/xiaohongshu.md`）配置文案格式
* **自动填充**：根据选中的记录，自动填充模板变量（如 `{{标题}}`, `{{我想表达}}`, `{{生词}}` 等）
* **文案预览**：提供文本框显示生成的文案，支持手动微调
* **一键复制**：提供按钮一键复制最终文案

#### 3️⃣ 小红书图片生成
* **功能同 v3.0**：
  * 风格选择 (Linear / Knowledge)
  * 4张卡片生成与预览
  * 单张下载与批量下载
  * 移动端显示长按保存提示，下载按钮可能不生效
  * 局域网手机测试可通过 URL 参数覆盖后端地址
  * 局域网手机测试允许后端返回局域网 Origin
* **尺寸**：1080 × 1440 (PNG格式)
* **背景留白**：左右 120px，上下 140px
* **安全区域**：内容区域最大宽度 840px

#### 风格系统详解

**1. Linear 风格 (极简产品风)**

* **定位**：极简、产品说明书、方法论
* **背景**：#FAFAFA
* **文字**：主文字 #111111，正文 #555555，强调色 #4F7FFF
* **字体**：中文 Noto Sans SC，日文 Noto Sans JP
* **字号**：标题 64px，副标题 42px，正文 36px，日语句子 38px，说明 28px，标签 24px
* **特殊元素**：
  * 左侧竖线：宽度 6px，颜色 #4F7FFF
  * 标签样式：背景 #EEF3FF，文字 #4F7FFF，圆角 12px

**2. Knowledge 风格 (知识卡片风)**

* **定位**：知识博主、学习卡片、易传播
* **背景**：#FFFFFF (辅助背景 #F6F6F6)
* **文字**：#222222，强调色 #FF7A45
* **卡片容器**：圆角 24px，阴影 10%，背景 #FFFFFF

#### 排版规则

* **日语句子**：自动换行，行宽最大 18 字符
* **避免**：超长行
* **间距**：段落间距 24px，模块间距 48px

### 5️⃣ 技术实现约束

* **前端**：简单 HTML / JS 页面（可用 Trae + Vibe Coding 快速搭建）
* **后端**：JS / Node.js 函数，负责：

  * 文本解析
  * Notion API 写入
* **图片生成方式**: HTML + CSS 渲染 → 截图生成图片，风格可控、生成稳定，无 AI 成本
* **部署**：Cloudflare Pages / Cloudflare Workers
* **安全性**：Token 隐藏或通过 Cloudflare Worker 处理 API请求
* **配置管理**：API URL和密钥通过配置文件管理，支持环境变量

### 页面 3：导出复盘 (export.html)

#### 1️⃣ 最近记录查询
* **默认展示**：进入页面不自动查询，点击“查询”后加载最近一周 Notion 记录
* **日期范围**：支持按日期范围查询
* **学习者昵称**：单行输入，浏览器记录当前昵称
* **学习者筛选**：仅查询“学习者”与昵称一致的记录
* **交互**：允许勾选多条记录，支持全选与反选
* **导航**：提供跳转回"输入与同步"与"查询与分享"页面的链接
* **视觉一致性**：配色、按钮、卡片、字体层级与其他页面统一
* **移动端便利**：表单与列表在手机清晰可点，按钮触达区域充足

#### 2️⃣ 复盘导出
* **导出格式**：点击导出后生成 markdown 文件
* **模板机制**：支持可定制模板文件（如 `templates/review.md`）
* **默认模板**：若无模板，按全部字段顺序生成
* **多选导出**：将选中的多条记录按顺序合并输出

## 三、全局视觉
* **站点图标**：三页面统一配置 favicon，保持品牌一致性

### 统一导航与页面布局
* **统一导航栏**：三页面固定在顶部，包含系统名称、功能导航、当前用户
* **导航高亮**：当前页面在导航中保持高亮状态
* **统一页面容器**：内容区域统一最大宽度，避免切换时宽度跳动
* **统一页面结构**：AppLayout → Navbar → PageContainer → PageHeader + PageContent
* **统一高度**：PageHeader 高度一致，PageContent 设定最小高度以避免页面跳动

## 四、非功能需求

1. 页面轻量、响应快速
2. 输入流程不超过 1 分钟完成（从粘贴到同步成功）
3. 图片生成小于 10 秒完成
4. 遵循 **日语真实表达系统 3.0** 冻结结构，字段不可新增
5. 保持可扩展性：后续可添加字段验证、批量输入或更多分享格式
