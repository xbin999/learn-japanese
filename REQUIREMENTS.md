# 📄 日语真实表达系统 — 页面输入同步工具需求文档 v3.0

## 一、目标

为日语真实表达成长系构建一个轻量输入工具，实现：

1. 用户将与 AI 讨论后的结构化文本直接粘贴到页面
2. 系统解析文本字段
3. 自动写入 **Notion "日语表达记录"数据库**
4. 支持可扩展的字段解析
5. 自动生成小红书分享图片

> 目标是保持操作简洁、轻量、快速完成沉淀，同时提供美观的分享图片生成功能。

---

## 二、页面功能需求

### 1️⃣ 输入区域

* **文本框**：多行输入，支持Ctrl+A全选，右上角提供"清除"按钮一键清空
* **提示文本**：可显示"请粘贴 AI 讨论后的结构化文本"
* **字段检查**：可选，解析后显示结构化解析结果，包括：标题、主题、我想表达、分享标题、学习总结、进化过程、最终定稿、核心结构、表达升级点、错误记录、生词
* **提交按钮**：触发解析和写入 Notion
* **用户体验**：提供操作提示，支持快捷键操作

### 2️⃣ 文本解析与预览逻辑

* **解析规则**：结构参见 `INTERFACE.md`
* **容错处理**：自动处理缺省字段、换行或多版本进化过程
* **即时预览**：解析后在页面显示便签预览，格式如下：
  * **基本信息**：标题、主题、分享标题、学习总结
  * **进化过程**：展示 V1/V2/V3 版本演变
  * **最终定稿**：展示最终润色版本
  * **核心结构**：展示本次核心语法结构
  * **表达升级点**：展示表达优化要点
  * **错误记录**：红色高亮显示错误名称、典型错误及正确模式
  * **生词**：绿色高亮显示词汇、读音、解释及例句

### 3️⃣ Notion 同步逻辑

* 调用 Notion API 创建新条目（database_id = 日语表达记录数据库）
* 填充字段结构参见 `INTERFACE.md`
* 成功后显示提示：`已成功同步到日语表达记录数据库！`

### 4️⃣ 小红书图片生成功能

* **生成按钮**：解析完成后显示"生成小红书图片"按钮
* **风格选择**：
  * 提供 `Linear` (默认/推荐) 和 `Knowledge` 两种风格选择
  * 支持实时预览切换效果
* **预览展示**：生成4张卡片预览，网格布局展示所有卡片
* **交互功能**：
  * 点击单张卡片可单独下载
  * 提供"保存所有卡片"按钮一键批量下载
  * 文件名格式：`日期_标题_卡片序号.png`
* **下载控制**：用户自主决定下载时机，不自动下载
* **生成性能**：< 2秒生成4张高质量卡片

#### 统一图片参数

* **尺寸**：1080 × 1440 (PNG格式)
* **背景留白**：左右 120px，上下 140px
* **安全区域**：内容区域最大宽度 840px

#### 风格系统详解

**1. Linear 风格 (极简产品风)**

* **定位**：极简、产品说明书、方法论
* **背景**：#FAFAFA
* **文字**：主文字 #111111，正文 #555555，强调色 #4F7FFF
* **字体**：中文 Noto Sans SC，日文 Noto Sans JP
* **字号**：标题 64px，副标题 42px，正文 36px，日语句子 38px，说明 28px，标签 24px
* **特殊元素**：
  * 左侧竖线：宽度 6px，颜色 #4F7FFF
  * 标签样式：背景 #EEF3FF，文字 #4F7FFF，圆角 12px

**2. Knowledge 风格 (知识卡片风)**

* **定位**：知识博主、学习卡片、易传播
* **背景**：#FFFFFF (辅助背景 #F6F6F6)
* **文字**：#222222，强调色 #FF7A45
* **卡片容器**：圆角 24px，阴影 10%，背景 #FFFFFF

#### 排版规则

* **日语句子**：自动换行，行宽最大 18 字符
* **避免**：超长行
* **间距**：段落间距 24px，模块间距 48px

### 5️⃣ 技术实现约束

* **前端**：简单 HTML / JS 页面（可用 Trae + Vibe Coding 快速搭建）
* **后端**：JS / Node.js 函数，负责：

  * 文本解析
  * Notion API 写入
* **图片生成方式**: HTML + CSS 渲染 → 截图生成图片，风格可控、生成稳定，无 AI 成本
* **部署**：Cloudflare Pages / Cloudflare Workers
* **安全性**：Token 隐藏或通过 Cloudflare Worker 处理 API请求
* **配置管理**：API URL和密钥通过配置文件管理，支持环境变量

---

## 三、非功能需求

1. 页面轻量、响应快速
2. 输入流程不超过 1 分钟完成（从粘贴到同步成功）
3. 图片生成小于 10 秒完成
4. 遵循 **日语真实表达系统 3.0** 冻结结构，字段不可新增
5. 保持可扩展性：后续可添加字段验证、批量输入或更多分享格式
