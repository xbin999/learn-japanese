# 页面解析接口需求说明

## 页面解析与 Notion 同步规则说明

---

## 一、输入格式规则

系统接收一整段结构化文本，格式遵循以下规则：

### 1️⃣ 顶层字段

| 字段     | 识别方式                  |
| ------ | --------------------- |
| 标题     | `标题：` 开头，直到换行         |
| 主题     | `主题：` 开头，直到换行         |
| 我想表达   | `我想表达：` 开头，直到换行         |
| 进化过程   | `——进化过程——` 与下一个大块之间   |
| 最终定稿   | `——最终定稿——` 与下一个大块之间   |
| 本次核心结构 | `——本次核心结构——` 与下一个大块之间 |
| 表达升级点  | `——表达升级点——` 与下一个大块之间  |
| 错误记录   | `——错误记录——` 与下一个大块之间   |
| 生词     | `——生词——` 与下一个大块之间   |
| 学习总结   | `——学习总结——` 与下一个大块之间   |
| 分享标题   | `——分享标题——` 到文本结束        |

---

## 二、进化过程解析规则

在“进化过程”块中：

* 按 `V1：` / `V2：` / `V3：` 分割
* 每个版本保留原始文本

解析为：

```
versions = {
 v1:"",
 v2:"",
 v3:"",
 v4:""
}
```
最终组合为一个字符串字段写入 Notion

---

## 三、错误记录解析规则

错误块中：

* 每个错误以 `[错误X]` 开头
* 每个错误内部字段：

| 字段     | 识别方式      |
| ------ | --------- |
| 错误名称   | `错误名称：`   |
| 典型错误说明 | `典型错误说明：` |
| 正确表达模式 | `正确表达模式：` |

解析为数组：

```js
errors = [
  {
    name: "...",
    description: "...",
    correctPattern: "..."
  }
]
```

最终写入 Notion：

* 推荐存为富文本 Markdown
* 或 JSON 字符串字段（后续扩展）

---

## 四、生词解析规则

生词块中：

* 每个单词以 `[单词X]` 开头
* 内部字段：

| 字段   | 识别方式    |
| ---- | ------- |
| 词汇   | `词汇：`   |
| 读音   | `读音：`   |
| 中文解释 | `中文解释：` |
| 例句   | `例句：`   |

解析为数组：

```js
vocab = [
  {
    word: "...",
    reading: "...",
    meaning: "...",
    example: "..."
  }
]
```

最终写入 Notion：

* 推荐存为富文本 Markdown
* 或 JSON 字符串字段（后续扩展）

---

## 七、标题格式规则

标题字段必须遵循以下格式：YYYY-MM-DD 主题类别｜具体标题，针对生成，对于页面无需解析。

示例：2026-03-01 事件说明｜银行提取日元存款

规则：

1. 日期为系统当前日期
2. 使用全角「｜」
3. 标题不超过20字
4. 不允许改变字段顺序

---

## 八、主题字段固定枚举

主题字段必须从以下固定分类中选择一个：

* 日程/时间表达
* 个人计划
* 事件说明
* 意见表达
* 感受表达
* 新闻转述
* 比较说明
* 原因说明
* 词义提问

不允许新增分类。针对生成，对于解析和写入 Notion 无要求。

---

## 五、容错规则

1. 若某字段缺失 → 写入空字符串
2. 若某错误块字段缺失 → 该字段为空
3. 若生词无例句 → 例句字段为空
4. 不因格式轻微偏差报错

---

# 二、📄 日语表达记录输出示例

下面是一份完整示例，符合上面约定的标准格式。

标题：娘と朝ジョギングした日
主题：事件说明

——我想表达——
今天早上和女儿一起慢跑，聊了一些学校的事情。

——进化过程——
V1：
今朝は娘とジョギングしました。そして、たくさん話しました。

V2：
今朝は娘と一緒にジョギングをして、そのあと色々なことを話しました。

V3：
今朝は娘と一緒にジョギングをしながら、学校のことや最近感じていることについてゆっくり話しました。

——最终定稿——
今朝は娘と一緒にジョギングをしながら、学校のことや最近感じていることについてゆっくり話す時間を持てました。とても穏やかな朝でした。

——本次核心结构——

1. 〜しながら（动作并行）
2. 〜について（话题展开）
3. 抽象感受表达（穏やかな朝）

——表达升级点——

1. 从简单并列句升级为动作并行结构
2. 从“たくさん話しました”升级为具体话题展开
3. 加入情绪性总结句，提高自然度

——错误记录——
[错误1]
错误名称：动词连接过于简单
典型错误说明：使用「そして」进行简单并列，表达偏初级
正确表达模式：使用「〜しながら」或从属结构增强自然度

[错误2]
错误名称：表达过于抽象
典型错误说明：使用「たくさん話しました」缺少具体内容
正确表达模式：加入具体话题（〜のことや〜について）

——生词——
[单词1]
词汇：穏やか
读音：おだやか
中文解释：平静的、温和的
例句：今日は穏やかな天気ですね。

[单词2]
词汇：感じる
读音：かんじる
中文解释：感受到
例句：最近、時間が早く過ぎると感じます。

——学习总结——
使用「〜しながら」结构自然表达两个动作的同时进行

——分享标题——
日语句子别老用「そして」｜一个结构让表达瞬间自然
