# 解析与同步测试用例

本文件用于设计解析器与同步流程的测试用例，覆盖输入、容错与输出结构。

---

## 一、解析器测试用例（JSON → 内部结构）

### 用例 1：完整 JSON 输入
输入：AI 输出 JSON（所有字段均有内容）
期望：
- 所有字段映射完整
- 进化过程按 v1→v4 顺序拼接
- errors/vocab 数组逐项映射

### 用例 2：缺失非必填字段
输入：
- 缺失 summary、shareTitle
- errors、vocab 为空数组
期望：
- summary、shareTitle 为空字符串
- errors/vocab 为 []
- 其他字段正常映射

### 用例 3：versions 部分缺失
输入：
- versions 仅包含 v1、v3
期望：
- 进化过程拼接为 v1 与 v3，保持顺序

### 用例 4：错误记录字段缺失
输入：
- errors 数组中对象缺少 description
期望：
- 对应“典型错误说明”为空字符串

### 用例 5：生词例句缺失
输入：
- vocab 数组中对象缺少 example
期望：
- 对应“例句”为空字符串

### 用例 6：非法 JSON
输入：
- 非 JSON 文本
期望：
- 抛出解析错误并提示原因

## 二、同步接口测试用例（内部结构 → /sync）

### 用例 7：最小可同步数据
输入：
- 仅标题、主题、我想表达
- 其他字段为空字符串或空数组
期望：
- /sync 返回 ok: true
- Notion 记录创建成功

### 用例 8：包含学习者昵称
输入：
- body.学习者 为非空字符串
期望：
- Notion “学习者”字段写入 select
